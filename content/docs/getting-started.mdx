# Getting Started

The docs are sourced from `content/docs` and compiled through `fumadocs-mdx` into the generated `.source` layer.

## Key files

| Path | Purpose |
| --- | --- |
| `source.config.ts` | Registers the docs collection and shared MDX config |
| `content/docs` | MDX content and navigation metadata |
| `src/lib/docs/source.ts` | Loader that maps generated docs content to `/docs` |
| `src/app/docs` | App Router layout and dynamic page renderer |
| `src/app/api/search/route.ts` | Search endpoint consumed by the docs UI |

## Add a new component page

1. Create a new `.mdx` file under `content/docs/components`.
2. Add it to `content/docs/components/meta.json` if you want explicit sidebar ordering.
3. Import a preview component or the component directly in MDX.
4. Restart the dev server if you add a brand new collection or change `source.config.ts`.

## Recommended pattern

Keep each component page focused:

- One short description of when to use it
- One or two interactive previews
- Prop notes or a manual prop table for public API details
- Implementation constraints only when they matter to consumers
